## Hexo 元数据

每篇文章添加完整 front matter：

- `title`：课程文章保留源文件的两位数字编号，学习路线不添加编号。
- `date` 与 `updated`：使用源文件的最后修改时间。
- `categories`：两级分类，分别为“研究生课程 / 电源硬件与数字电源”和“研究生课程 / 微波工程与电磁场”。
- `tags`：仅添加与文章主题直接相关、具有检索价值的少量标签。
- `permalink`：使用稳定英文路径，如 `/courses/power-electronics/03-bjt/`，避免中文 URL 编码。
- `description`：从文章主题生成一句简短摘要。

Obsidian callout、公式、表格和图片语法会转换为 Hexo 可渲染格式。文章末尾添加课程内“上一篇 / 下一篇 / 返回课程目录”导航，不依赖全站文章时间顺序。

## Courses 导航页

`/courses/` 只展示课程内容，不展示原始文件、图片、PDF、文件大小或下载链接。

页面包含两个课程区块：

1. 电源硬件与数字电源：课程简介、文章数量、学习路线、01–03 课程文章。
2. 微波工程与电磁场：课程简介、文章数量、严格按 00–17 排序的课程文章。

每个目录项直接链接到渲染后的博客文章。

## 图片整理规则

- 每张图片只保留一份博客副本。
- 按文章和出现顺序重命名，不保留 `Pasted image 时间戳.png`。
- 将所有 `![[...]]` Obsidian 内链改成以 `/img/courses/...` 开头的标准 Markdown 图片链接。
- 图片紧跟其解释段落，不连续堆叠无说明图片。
- 不改变图片画面内容；仅整理文件名、目录、引用、替代文字和图注。

## 错误处理

- 源文章引用的图片找不到时，停止该文章迁移并报告缺失文件，不生成失效链接。
- 出现同名图片时以内容哈希确认是否重复；不同内容不得覆盖。
- Hexo 无法解析的 Obsidian 语法逐项转换，不直接删除有意义内容。
- 已存在的博客文件若与迁移目标冲突，以本设计中的标准文章结构替换现有原始下载副本。
